NotePic 1.2.0: limpia imagenes locales de Obsidian despues de subirlas a OSS

Producto: NotePic-OSS

Version: 1.2.0

Fecha de publicacion: 17 de junio de 2026

Ultima actualizacion: 17 de junio de 2026

NotePic para Obsidian ya publico la version 1.2.0.

Muchos usuarios de Obsidian se encuentran con el mismo problema de flujo:

  1. Capturas, ilustraciones y adjuntos se guardan en el Vault mientras escriben
  2. NotePic sube esas imagenes a OSS antes de publicar
  3. Las referencias de imagen en Markdown pasan a ser enlaces en linea
  4. Los archivos locales originales siguen en la carpeta de adjuntos

Si solo escribes unas pocas notas, no es un gran problema.

Pero si usas Obsidian para escribir articulos, mantener una base de conocimiento o sincronizar todo el Vault con Git, esos restos locales se convierten poco a poco en un costo real.

Los archivos Markdown suelen pesar solo unos pocos KB. Lo que vuelve pesado a un repositorio suelen ser capturas, portadas, imagenes de demostracion y adjuntos que ya no estan referenciados por ninguna nota.

Cuanto mas pesado es el repositorio Git, mas se ven afectados clone, pull, push, las copias de seguridad y la sincronizacion entre dispositivos. Cuando un Vault de Obsidian acumula cada vez mas imagenes, es dificil recordar cuales siguen referenciadas y cuales son solo restos historicos despues de la subida.

Por eso la version 1.2.0 no agrega otro boton de subida. Hace que las imagenes locales vuelvan a ser visibles, revisables y limpiables.



Que trae NotePic 1.2.0

NotePic 1.2.0 agrega limpieza de activos de imagen locales.

Despues de esta actualizacion, puedes escanear imagenes locales dentro de un Vault de Obsidian y revisar si todavia estan referenciadas por notas.

La funcion cubre estas acciones:

  • Escanear archivos de imagen locales en el Vault
  • Separar imagenes referenciadas y no referenciadas
  • Contar imagenes y calcular el tamano total
  • Revisar la distribucion de imagenes por estructura de carpetas
  • Seleccionar imagenes locales que ya no son necesarias
  • Confirmar antes de eliminar

El objetivo es concreto: despues de subir imagenes a OSS y reemplazar las referencias de las notas con URLs en linea, puedes encontrar con mas seguridad los archivos locales que ya no estan referenciados por ninguna nota.



Por que importa para usuarios de Git

Si tu Vault de Obsidian solo vive en una maquina local, tener mas imagenes ocupa principalmente espacio en disco.

Si gestionas el Vault con Git, las imagenes entran en otro tipo de problema.

Git funciona muy bien para notas Markdown. Los archivos de texto son pequenos, las diferencias son claras, el historial es rastreable y la sincronizacion entre dispositivos sigue siendo transparente.

Git no esta pensado para acumular imagenes sin limite.

Una sola captura puede pesar mas que decenas de notas. Peor aun, borrar una referencia de imagen dentro de una nota no siempre elimina el archivo adjunto. Con el tiempo, el repositorio puede conservar muchos archivos que ya no usas pero que siguen bajo seguimiento de Git.

Aqui es donde NotePic 1.2.0 ayuda: conecta la limpieza con el flujo posterior a subir imagenes a OSS.

Un flujo de publicacion mas ligero puede quedar asi:

  1. Escribes normalmente en Obsidian
  2. Mantienes las imagenes locales mientras escribes
  3. Subes imagenes a OSS con NotePic antes de publicar
  4. Confirmas que las referencias de las notas ya son URLs en linea
  5. Escaneas las imagenes locales del Vault
  6. Limpias las imagenes que ya no estan referenciadas
  7. Haces commit en Git

Asi el repositorio Git conserva principalmente notas y activos que todavia deben quedarse en local, no una coleccion de archivos que perdieron su relacion de referencia despues de la subida.



La limpieza debe ser cuidadosa, no agresiva

La limpieza de imagenes no deberia ser demasiado agresiva.

"No referenciada" no siempre significa "seguro para borrar ahora mismo".

Algunas imagenes pueden no estar referenciadas por los Markdown actuales, pero seguir siendo utiles como material fuente. Otras pueden ser usadas por plugins especiales, Canvas, fragmentos HTML u otro flujo de trabajo. Algunos articulos antiguos pueden no haber terminado su migracion.

Por eso el objetivo de NotePic 1.2.0 no es fomentar una eliminacion ciega con un clic. Limpiar imagenes no se trata de forzar que el Vault no tenga imagenes locales. Se trata de ayudarte a entender que imagenes todavia necesitan quedarse en local.



Cierre

Si solo lees notas locales, no publicas y no sincronizas con Git, puede que todavia no necesites esta funcion.

Pero si ya usas NotePic para subir imagenes, o si las carpetas de adjuntos de Obsidian y el tamano del repositorio Git empiezan a molestarte, vale la pena actualizar a la version 1.2.0.
