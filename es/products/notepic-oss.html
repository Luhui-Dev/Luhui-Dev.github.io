NotePic-OSS

Abrir NotePic-OSS

Problema

Markdown y Obsidian son comodos para escribir porque permiten concentrarse primero en el texto. La parte molesta suele llegar despues: el texto se copia facil, pero las imagenes no siempre lo siguen.

Las referencias locales suelen verse asi:

![[Pasted image 20260603102000.png]]
![](./images/cover.png)
<img src="./img/screenshot.png" />

Funcionan dentro de Obsidian, Typora, VS Code o un repositorio local de blog. Pero al copiar el mismo articulo a WeChat, Zhihu, Juejin, un blog personal u otro sistema de publicacion, las rutas se vuelven inestables. La plataforma no puede leer adjuntos locales, las rutas relativas pueden romperse y una migracion puede dejar el texto intacto pero las imagenes perdidas.

NotePic-OSS resuelve ese paso de limpieza de imagenes antes de publicar.

Que hace

NotePic-OSS sube las imagenes locales de un articulo a Aliyun OSS y reemplaza los enlaces originales por URLs remotas.

Antes:

![[architecture.png]]
![](./assets/demo.png)

Despues:

![](https://your-cdn.example.com/markdown/architecture.png)
![](https://your-cdn.example.com/markdown/demo.png)

Despues de eso, el Markdown puede copiarse entre plataformas y las imagenes siguen siendo URLs alojadas, no archivos locales.

Se enfoca en las partes que suelen fallar en sistemas de contenido:

  • Analizar referencias de imagen en Markdown, MDX y HTML
  • Subir imagenes locales a Aliyun OSS
  • Comprimir recursos compatibles antes de subirlos
  • Usar deduplicacion por hash para evitar subidas repetidas
  • Reemplazar rutas locales por URLs remotas de OSS
  • Reconocer formatos comunes como wikilinks de Obsidian, imagenes Markdown e imagenes HTML

Dos formas de usarlo

  • Plugin de Obsidian: procesa las imagenes de la nota actual antes de copiar el articulo a plataformas de publicacion.
  • CLI: procesa Markdown, MDX o HTML en proyectos de ingenieria, repositorios de blog, CI, migraciones o flujos por lote.

El objetivo es directo: convertir la limpieza repetida de imagenes antes de cada publicacion en un clic o un comando.

Estado actual

El proyecto es gratuito, open source y se mantiene como herramienta de laboratorio. Actualmente prioriza Aliyun OSS porque es el flujo real que uso cada dia.

Entradas principales:

  • Pagina del proyecto: NotePic-OSS
  • Repositorio: Luhui-Dev/notepic-oss

Siguiente paso

La direccion mas util es seguir mejorando el flujo real de publicacion: documentacion de instalacion y configuracion mas clara, mejores ejemplos para Obsidian y CLI, y expansion de proveedores de almacenamiento segun feedback, incluyendo Tencent Cloud COS, S3, Qiniu Cloud o UpYun.
