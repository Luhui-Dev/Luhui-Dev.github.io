Algeo SDK 2.6.0 上線：讓幾何能力更穩地進入你的產品

產品: 大角幾何畫板

版本: SDK 2.6.0

發布日期: 2026年5月19日

最後修改: 2026年5月19日

我們剛剛發布了 Algeo SDK 2.6.0。

Algeo SDK 是大角幾何對外提供的開放能力之一。我們希望它不是一個孤立的編輯器，而是一套可以被教育產品、AI 數學產品、題庫系統和教研工具接進去的幾何能力：能畫、能展示、能互動，也能進入你自己的內容和業務流程。

過去一段時間，我們和一些教育產品、AI 數學產品、題庫系統的團隊交流時，聽到的回饋很樸素。

大家並不是只問 SDK 有哪些 API。更多時候，他們問的是：

  • 老師畫好的圖，能不能直接放到課件、講義和試卷裡？
  • 學生打開題目時，畫板會不會偶爾卡住？
  • AI 生成的幾何步驟，能不能匯出成解析圖或報告圖？
  • 同一份幾何素材，能不能既能互動，又能沉澱為平台裡的內容資產？
  • 接入過程中遇到異常，SDK 能不能自己處理得更乾淨一點？

這些問題聽起來都不複雜，但對一個要上線的產品來說，每一個都很重要。

所以這個版本，我們主要做了兩件事：增加畫板圖片匯出能力，以及提升嵌入初始化的穩定性。同時，我們也把範例和文件補了一輪，希望開發團隊在本地除錯、寫 demo、做上線前驗證時，能更順一些。

如果你想直接試，可以使用 @dajiaoai/algeo-sdk@2.6.0。圖片匯出的範例案例也已經放出來了：Export Slide Image Demo。

如果你們的產品裡有幾何題、幾何課件、AI 解題、題庫編輯、作業報告或教研內容生產，下面這些場景應該會比較熟悉。



場景一：老師做完動態幾何，也能一鍵進入課件和講義

很多教育產品都希望給老師提供互動式幾何畫板。

但老師真正的工作流並不只停在畫板裡。她可能要把圖放進 PPT，放進講義，放進試卷，放進課後作業解析，甚至放進公眾號或校本資源庫。

過去，動態圖和靜態內容之間經常隔著一道牆：畫板裡能動，但匯出時又要截圖；截圖能用，但清晰度、尺寸、批次處理和重複使用都不穩定。

Algeo SDK 2.6.0 新增了畫板圖片匯出能力，讓產品團隊可以把幾何畫板裡的內容匯出為圖片，並進一步流轉到自己的課件、講義、試卷、報告和內容管理系統中。

這意味著老師在你的產品裡完成的幾何素材，不再只是一個「線上畫板狀態」，也可以變成一份可發布、可歸檔、可傳播的教學資產。

對教育產品團隊來說，這個能力非常實際：

  • 課件工具可以把幾何圖直接插入幻燈片
  • 題庫系統可以生成標準題圖和解析圖
  • 教研平台可以把動態素材沉澱為靜態預覽
  • 作業系統可以在報告裡展示關鍵圖形
  • 內容團隊可以把同一份幾何素材用於多端發布

幾何內容從此不只是「能互動」，也更容易進入真實的內容生產鏈路。

我們也準備了一個可用的匯出圖片範例，方便你先看效果再評估流程：12-export-slide-image.html。



場景二：題庫和作業系統，需要更穩定的嵌入體驗

題庫、作業和測評系統對穩定性的要求很高。

學生打開一道題時，幾何圖不能偶爾載入失敗；老師批改作業時，畫板不能卡在初始化狀態；運營人員配置內容時，也不能靠重新整理頁面來賭一次成功。

Algeo SDK 2.6.0 對嵌入初始化流程做了穩定性提升，尤其是針對 iframe 載入、逾時處理、異常清理等接入過程中常見的問題。

這些變化對最終使用者來說可能不顯眼，但對產品團隊很重要。

因為在真實產品裡，一個幾何元件不是只在範例頁面打開一次，而是會出現在大量頁面、不同網路、不同裝置、不同權限狀態和複雜的單頁應用路由中。

更穩定的嵌入能力，意味著：

  • 學生端打開題目時，幾何內容載入更可預期
  • 老師端編輯題目時，異常狀態更少打斷工作
  • 平台方排查問題時，鏈路更清晰
  • 產品可以更放心地把幾何元件放到核心流程裡

對教育平台來說，穩定不是錦上添花，而是幾何能力能不能進入主流程的前提。



場景三：AI 數學產品，不只需要解出來，還要呈現出來

AI 數學產品正在從「給答案」走向「給過程」。

尤其是幾何題，學生真正需要的往往不是一句「連接某點並作輔助線」，而是看到這條線為什麼要畫、畫完之後關係如何變化、不同構型之間有什麼聯繫。

當 AI 能生成幾何過程，下一步就會遇到內容呈現的問題：

  • 生成的圖能不能穩定嵌入在解題頁？
  • 多步講解能不能拆成不同畫板或步驟？
  • 每一步能不能匯出為圖片，用在解析、報告或分享頁裡？
  • 學生和老師能不能在同一個幾何內容上繼續編輯、複盤和二次創作？

Algeo SDK 2.6.0 對「嵌入可靠性」和「圖片匯出」的增強，正好補上了這條鏈路。

它讓 AI 生成的幾何內容，不只停留在一次性的頁面展示裡，而是可以進入更完整的產品閉環：生成、展示、互動、匯出、沉澱、複用。

對 AI 數學產品來說，這會直接影響使用者感知。

因為使用者不會關心背後呼叫了什麼模型，也不會關心畫板是怎麼初始化的。他們只會感受到：這道幾何題講得清不清楚，圖能不能打開，步驟能不能看懂，內容能不能帶走。



場景四：教研和內容團隊，需要把幾何素材變成可管理資產

在很多學校、機構和內容公司裡，幾何素材長期是「散落的檔案」。

一部分在老師電腦裡，一部分在 PPT 裡，一部分是截圖，一部分是舊軟體原始檔案，還有一部分在題庫系統裡但無法重複使用。

這會帶來一個很現實的問題：幾何內容很難資產化。

Algeo SDK 的價值，不只是把一個畫板嵌進產品，而是把大角幾何已經累積下來的動態幾何能力開放出來，讓幾何素材可以納入你自己的業務系統：

  • 誰建立了這份素材
  • 屬於哪個年級、章節、知識點
  • 哪些老師可以編輯
  • 哪些內容可以發布給學生
  • 哪些圖可以匯出到講義和試卷
  • 哪些素材可以進入 AI 生成或推薦流程

2.6.0 對匯出和嵌入鏈路的增強，會讓這套資產化過程更順。

內容團隊可以在自己的 CMS 裡建立幾何圖，匯出標準圖片用於出版和分發；教研平台可以為每份動態幾何素材生成預覽圖；題庫系統可以讓同一份內容同時服務編輯端、學生端和列印端。

幾何不再是產品之外的「配圖工作」，而是可以進入平台資料結構和內容流程的一等資產。



場景五：產品團隊真正想要的，是少操心

對一個準備上線的產品團隊來說，選擇一個 SDK，表面上是在選擇能力，實際上是在選擇一段長期合作關係。

你當然會關心它能不能畫點線圓、能不能編輯、能不能展示、能不能和業務系統通訊。

但上線之後，你更關心的是：

  • 使用者打開時是否穩定
  • 出問題時是否能被正確處理
  • 範例和文件是否跟得上
  • 新同事接手時是否容易理解
  • 未來版本是否會持續補齊真實場景裡的缺口

Algeo SDK 2.6.0 同步更新了範例、設定支援和文件，讓接入體驗更貼近真實專案，而不是只服務一個最簡單的 demo。

這件事很小，也很關鍵。

因為 SDK 不是給技術同學「看一眼」的，它最終要進入一個產品團隊的協作流程：產品經理要確認場景，前端要接入元件，後端要處理資料歸屬，運營要配置內容，客戶成功要解釋效果。

文件和範例越接近真實流程，產品團隊走到上線的路就越短。



Algeo SDK 2.6.0 適合哪些團隊關注

如果你的產品裡已經出現以下任意場景，就值得了解 Algeo SDK：

  • 線上教育平台：需要在課程、題庫、作業或課堂工具中加入動態幾何能力
  • AI 數學產品：希望把幾何題從文字解析升級為視覺化、可互動的解題過程
  • 題庫和測評系統：希望幾何題不再只是靜態圖片，而是可編輯、可展示、可沉澱的資料
  • 教研協作平台：希望老師共建、重複使用和管理幾何教學素材
  • 教輔出版和內容生產團隊：希望提高幾何配圖、課件圖、講義圖和試卷圖的生產效率
  • 教育 SaaS 廠商：希望低成本補齊專業幾何能力，而不是從零開發一套畫板

一句話：只要你的產品需要處理「幾何內容」，Algeo SDK 都可以把大角幾何的開放能力接入到你的底層流程裡。



寫在最後

Algeo SDK 2.0 讓幾何畫板可以嵌進產品裡。

Algeo SDK 2.6.0 則讓這件事更進一步：嵌入更穩定，內容更容易匯出，範例和文件更貼近產品團隊的真實工作。

這不是一次面向技術參數的更新，而是一次面向產品落地的更新。

因為教育產品真正需要的不是一個孤立工具，而是一種可以進入業務流程的幾何能力：能被老師使用，能被學生打開，能被內容團隊複用，能被 AI 生成，能被平台管理，能在關鍵時刻穩定工作。

Algeo SDK 2.6.0 已正式上線。

如果你正在為自己的教育產品補齊幾何能力，現在是一個很適合開始接入的版本。接入時可以直接使用 @dajiaoai/algeo-sdk@2.6.0。
